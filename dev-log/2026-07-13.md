## Bug 修复记录 (调试阶段)

### 1. AndroidManifest 缺 INTERNET 权限
- 现象: `SecurityException: Permission denied (missing INTERNET permission?)`
- 修复: `AndroidManifest.xml` 加 `<uses-permission android:name="android.permission.INTERNET"/>`

### 2. Navigation 路径 / 符号冲突
- 现象: `NullPointerException: getString on null Bundle` → 闪退
- 根因: 文件路径 `/data/user/0/.../cache/xxx.jpg` 中的 `/` 被 Navigation 当路由分隔符解析
- 修复: `Uri.encode(path)` 传出, `Uri.decode()` 还原

### 3. CameraScreen/ResultScreen ViewModel 不同实例
- 现象: ResultScreen 显示"暂无分析结果"
- 根因: 两个 Screen 在不同 NavBackStackEntry 中，`hiltViewModel()` 各自创建新 ViewModel
- 修复: 改为 `hiltViewModel(LocalContext.current as ComponentActivity)` 提升到 Activity 级别共享

### 4. 模型名称错误 — qwen-plus 不支持图片
- 现象: API 200 但返回"未提供实物照片，无法识别菜品信息"
- 根因: `qwen-plus` 是纯文本模型，需要 `qwen-vl-plus` 视觉-语言模型
- 修复: `FoodAnalysisRepositoryImpl` model 改为 `qwen-vl-plus`

### 5. 浮动拍照按钮与导航栏按钮重叠
- 修复: 删除 EatmateNavGraph 中浮动的 Box(CameraNavButton) 组件

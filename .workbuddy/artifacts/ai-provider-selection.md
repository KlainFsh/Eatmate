## 二、逐家分析

### 2.1 百度 AI 菜品识别 ⭐⭐⭐⭐⭐ (推荐)

**产品**: [图像识别 - 菜品识别](https://ai.baidu.com/tech/imagerecognition/dish)

| 指标 | 详情 |
|------|------|
| **中餐覆盖** | 9000+ 种菜品，含地方特色菜，支持自建图库 |
| **识别准确率** | 公开数据 95%+ |
| **返回热量** | ✅ **直接返回热量 (calorie/100g)** |
| **多菜品识别** | 标准版单个菜品; **自定义菜品识别** 支持多菜品并发识别 |
| **返回数据** | 菜名、置信度、热量、百科信息 |
| **免费额度** | 个人 1 万次/年; 企业 3 万次/年 |
| **预付费包** | 1万次/29元 → 0.0029元/次 |
| **后付费** | 0.3元/千次(百万级) → 0.0003元/次 |
| **QPS 限制** | 免费 2 QPS，付费 10 QPS(可扩) |
| **Android SDK** | ✅ 官方 Java SDK ([Baidu-AIP/Java-SDK](https://github.com/Baidu-AIP/java-sdk)) |

**API 返回示例**:
```json
{
  "result": [
    {
      "name": "烤鸭",
      "probability": "0.988",
      "has_calorie": true,
      "calorie": "530"
    }
  ]
}
```

> 关键点: `has_calorie=true` + 直接给 `calorie` 值，省去一次 API 调用，对智食的核心场景完美契合。

**自定义菜品识别扩容性**: 用户可上传自有菜品图片训练专属模型，适合后续高频外卖/食堂菜品定制化。

### 2.2 阿里云 菜品识别 ⭐⭐⭐

**产品**: [视觉智能开放平台 - RecognizeFood](https://vision.aliyun.com/)

| 指标 | 详情 |
|------|------|
| **中餐覆盖** | 官方未公布具体数量，实测支持常见中餐 |

| **多菜品识别** | ❌ 单图单菜品 |
| **免费额度** | 存在但具体额度未知 |
| **价格** | 未查到公开价格文档 |
| **Android SDK** | 无官方 SDK，需用阿里云 OpenAPI 通用 SDK |
| **API 版本** | 2019-09-30，**多年未更新** |

> 核心问题: API 停留在 2019 年，不返回热量数据，需要额外调用营养数据库。产品迭代停滞，不建议选。

### 2.3 腾讯云 ⭐⭐

| 指标 | 详情 |
|------|------|
| **中餐覆盖** | 无专门的菜品识别产品 |
| **替代方案** | 图像分析(无菜品专类) + 混元多模态大模型 |
| **返回热量** | ❌ 图像分析不返回; 混元可通过 prompt 设计实现 |
| **免费额度** | 混元有免费 token (但按 token 计费，图片成本高) |
| **价格** | 混元多模态按 token 计费，单张图片约 0.01-0.05 元 |
| **Android SDK** | 无独立菜品 SDK |

> 核心问题: 腾讯云没有成熟的中餐识别产品。用混元大模型"曲线救国"：需精心设计 prompt 提取菜名+热量，结果不结构化，稳定性差，单次成本高。

## 四、结论: 推荐 百度 AI 菜品识别

### 选择理由

1. **热量直出是杀手锏**: 智食的核心是"拍照 → 热量分析"，百度返回 `calorie` 字段省掉了一步"菜名 → 营养库匹配"的延迟，真正做到 3 秒闭环。

2. **9000+ 种中餐覆盖**: 中文菜品识别领域深耕多年，覆盖炒菜、蒸菜、火锅、面点、地方菜等，是国内中餐识别当之无愧的第一。

3. **成本极低**: 免费 3 万次（企业认证）够 MVP 测试全部调用；商用后 29 元/万次，折合 0.0029 元/次。就算日活 1 万人每人拍照 3 次，月费也就约 260 元。

4. **自定义图库**: 后续可为外卖爆款、食堂档口等自建图库，准确率还能再拉一截。

5. **Java SDK**: 直接集成到 Android 项目，无需自己封装 HTTP 请求。

### 风险与兜底

| 风险 | 兜底方案 |
|------|----------|
| 外卖混合菜品识别不准 | 自定义菜品图库 + 手动修正入口 |
| 部分菜品无热量数据 (`has_calorie=false`) | 降级到本地 Room 营养库匹配 |
| API 偶发超时 (5G/4G 切换) | 3 秒超时 + offline queue 重试 |
| 隐私合规 (图片上传) | 仅上传识别结果，原图本地存储 + 隐私政策声明 |

### 集成架构

```

             │   智食 Android   │

             └─────────────────┘
```

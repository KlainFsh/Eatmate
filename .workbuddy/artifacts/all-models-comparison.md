## 四、细评每个模型

### 4.1 🥇 Qwen3.6-Plus — 强烈推荐

```
价格: 输入 0.8元/千tokens (图文统一) | 输出 2.0元/千tokens
千次成本: ≈2.16元
免费额度: 100万 tokens (新用户, 180天有效)
```

**为什么它是最佳选择**：

1. **成本最低** — 千次调用 2.16 元，同样钱在 GLM-4V 上只能跑 60 次
2. **能力最新** — 2026 Qwen3 系列，支持 1M 上下文 + 函数调用 + 多模态输入
3. **图文统一计费** — 图片和文本同样的费率，不像 Qwen3-VL-Plus 图片价格是文本的 2 倍
4. **阿里官方推荐** — "效果/速度/成本均衡的推荐模型，适用于大多数通用场景"
5. **国内原生** — 阿里云百炼直连，不需要任何代理或中转
6. **食物领域积累** — 阿里+复旦 FoodLMM 研究成果加持

**唯一需要验证的**: 食物识别准确率。但 Qwen 系列的中文多模态能力在 MMBench、MME 等 benchmark 上均接近国际第一梯队，食物识别作为视觉理解的子集没有理由差。

### 4.2 🥈 Gemini 2.5 Flash — 能力最强但不方便

```
价格: 输入 $0.30/1M | 输出 $2.50/1M
千次成本: ≈$1.39 (约¥10, 含中转加价)
免费额度: ~50次/天
```

**优势**: 多模态排行榜常驻第一，识别最准确，免费层够 MVP 测试。

**致命问题**: 中国大陆无法直连。需要通过 Vertex AI（香港/新加坡节点）或第三方 API 中转平台。**中转意味着增加延迟、降低稳定性、加价 20-50%。**

**结论**: 如果你愿意接受中转方案，Gemini 2.5 Flash 的能力最强。但 Qwen3.6-Plus 在中文食物场景下大概率不差，且原生可用。

### 4.3 🥉 GPT-4o / gpt-5.4-mini — 能力强但贵

```
价格: 输入 $0.15/1M | 输出 $0.60/1M (gpt-4o-mini 已升为 gpt-5.4-mini)
千次成本: ≈$1.83 (约¥13)
```

比 Qwen 贵 6 倍且同样需要中转，没有选的理由。

### 4.4 ❌ GLM-4V — 太贵了

```
价格: 输入 ¥50/1M | 输出 ¥50/1M
千次成本: ≈¥35
```

**是 Qwen3.6-Plus 的 16 倍价格**。能力相当，价格高 16 倍，无法推荐。

### 4.5 ❌ DeepSeek-VL2 — 没有商业 API

DeepSeek 目前只提供文本模型 API。VL2 是开源模型但没有商业托管服务，需要自己部署——MVP 阶段不考虑。

### 4.6 ❌ 豆包 Vision — 能力不明确

字节豆包主要在文本领域发力，视觉模型能力未公开 benchmark 数据，且 API 文档中未明确食物识别支持。风险太大。

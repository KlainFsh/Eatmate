## 智食 (Eatmate) 项目开发前评估

- 分析了现有项目状态（Android 基础模板，Kotlin/Compose/M3）
- 输出完整开发前建议文档 `pre-dev-advice.md`
- 关键决策建议：
  - minSdk 从 30 降为 26（覆盖 97% 中国用户）
  - 架构：MVVM + Clean Architecture，Hilt + Navigation Compose + Room + Retrofit + CameraX
  - AI 识别策略：云端 API 优先，后续可考虑端侧部署
  - 品牌色方案：健康绿 + 暖橙（Health/Wellness 风格）
  - 分三阶段开发：MVP(4-6周) → 核心完善(4-6周) → 辅助功能(4周)
  - 中国市场适配：微信登录、厂商推送、中餐营养数据库(3000+)、外卖/食堂场景
- 待确认：AI 识别服务商选择（百度AI/腾讯云/阿里云）
- 完成 AI 服务商选型调研，**推荐百度 AI 菜品识别**:
  - 唯一直接返回热量数据的服务商 (核心杀手锏)
  - 9000+ 中餐覆盖，95%+ 准确率
  - 企业认证 3 万次免费 + 29 元/万次商用
  - 阿里云 API 多年未更新，无热量数据
  - 腾讯云无专门菜品识别产品，混元方案成本高不稳定
- 完成百度 API 文档深度解析 (`baidu-api-analysis.md`)，发现关键问题：
  - **标准 v2/dish 只支持单菜品识别**，多菜品场景需用"自定义菜品识别"API
  - `calorie`/`probability` 返回类型是 String 不是 Number，需要 toFloat()
  - 热量仅是每100g数据，需自建份量估算模块
  - 需配套本地 Room 营养库（蛋白/碳水/脂肪），API 不返回这些
  - 建议 filter_threshold=0.85 + 手动修正入口
  - access_token 30天有效，用 DataStore 缓存+自动刷新
  - 自封装 Retrofit 替代官方 Java SDK
- 最终 AI 方案: 阿里百炼 Qwen3.6-Plus
  - 费用: 0.8元/千tokens(图文统一), 千次分析≈2.16元, 千DAU月费≈6元
  - 优势: 图文统一低价、1M上下文、Function Calling、国内直连、复旦FoodLMM学术背景
  - 淘汰方案: 百度v2/dish(单菜品数据不全)、GLM-4V(贵16倍)、GPT-4o(需中转且贵6倍)、DeepSeek-VL2(无商业API)
  - Gemini 2.5 Flash 作为可选的最高能力兜底(需第三方中转)

## 执行规则

1. **严格按 Phase 顺序执行** — 不跳步，不并行跨 Phase
2. **每个 Phase 完成后确认** — 用户验收通过再进下一 Phase
3. **每天收工前更新 dev-log** — `dev-log/YYYY-MM-DD.md`
4. **每次 git 提交粒度** — 一个任务一个 commit
5. **编译不过不提交** — 每次都先跑 `./gradlew assembleDebug`

> 下一个行动: 开始 Phase 1-1，引入所有基础依赖。

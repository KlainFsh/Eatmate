# 智食 产品需求文档

> 版本: 1.0-MVP | 更新: 2026-07-03

---

## 一、产品定义

**智食** 是一款垂直面向减脂/健身人群的 AI 饮食管理 App。用户拍照即可自动识别菜品，获取热量、蛋白质、碳水、脂肪等宏量营养素分析。

### 目标用户

- **主要**: 白领（外卖高频）、学生（食堂/校园餐）
- **次要**: 健身人群（自己做饭/备餐）

### 核心痛点

- 记饮食太麻烦 — 手动搜菜名、填重量，3 分钟入门 3 天放弃
- 外卖/食堂热量无法知晓 — 外面的菜不知道多少油
- 营养无法量化 — 知道"吃多了"但不知道蛋白质够不够

## 二、MVP 功能范围 (Phase 1-4)

### 2.1 拍照识别 (P0 — 核心)

```
用户拍一张食物照片
    → AI 自动识别所有菜品
    → 显示: 菜名、热量(kcal)、蛋白质(g)、碳水(g)、脂肪(g)、预估重量(g)
    → 用户可确认或修改
    → 保存到今日饮食记录
```

**验收标准**:
- 拍照到出结果 ≤ 3 秒
- 6 种测试场景至少 5 种菜品名正确
- JSON 格式 100% 干净无 markdown 包裹

### 2.2 饮食日记 (P0)

```
按日期查看饮食记录
    → 三餐分类: 早餐/午餐/晚餐/加餐
    → 每日营养汇总: 总热量/蛋白/碳水/脂肪
```

**验收标准**:
- 能查看今天/昨天/前天的记录
- 日汇总数字与餐食明细总和一致

### 2.3 用户目标 (P1)

```
设定减脂/增肌目标
    → 输入: 当前体重、目标体重、每周目标
    → 日摄入与目标对比展示
```

**验收标准**:
- 可选择 减脂/增肌/维持 三种模式
- 日记页显示 "1250 / 目标 1800 千卡"

### 2.4 不做的事 (MVP 不做)

- ❌ 一键定制三餐 (Phase 5+)
- ❌ 周/月趋势分析 (Phase 5+)
- ❌ 社交分享
- ❌ 微信登录 (先用设备本地)
- ❌ 推送通知
- ❌ 搜索/手动录入食物
- ❌ 离线识别

## 四、核心交互流程

### 主流程: 拍照 → 分析 → 记录

```
[打开 App]
    ↓
[相机预览页]  ← 默认首页, 大快门按钮置底
    ↓ 点击快门
[拍照]  →  图片压缩(≤512KB)
    ↓
[加载中]  "AI 在分析中…" + 骨架脉冲动画
    ↓ 3 秒内
[结果页]
    ├── 拍照缩略图
    ├── 菜品列表 (名称 + 置信度)
    ├── 营养数据 (热量大字 + 蛋白质/碳水/脂肪 Chip)
    └── [记录这顿饭] 按钮
        ↓ 点击
[保存成功] → 跳转日记页
```

## 五、关键数据指标 (后续埋点)

| 指标 | 定义 | 目标 |
|------|------|------|
| 拍照到结果耗时 | 快门 → 结果页完全渲染 | < 3 秒 |

| 日活留存 | 第二天回来的用户比例 | > 40% |
| 崩溃率 | Crash-free session rate | > 99.5% |
